# Requirements

## Functional
- R1: System shall accept an authentication attempt from Android over Bluetooth
- R2: System shall unlock only on valid authentication
- R3: System shall relock automatically after a configurable time
- R4: System shall provide user feedback (success/failure)
- R5: System shall store credentials persistently (not plain-text)

## Non-functional
- N1: Max unlock time shall be limited for safety
- N2: Firmware shall recover from communication errors without locking open
- N3: Repo shall include documentation, wiring/PCB files, and build instructions
